# Agents

Salesforce CLI plugin for managing pools of pre-created scratch orgs (`sf pool <subcommand>`).

For coding conventions scoped by file type, see `.github/instructions/`.
For architecture and lifecycle details, see `.github/copilot-instructions.md`.

## Commands

- `sf pool prepare` — create scratch orgs and tag them into a pool
- `sf pool fetch` — find an available org and mark it assigned
- `sf pool list` — query and display pool org counts by status
- `sf pool clean` — delete failed/stale/expired orgs from a pool

## Build and test

```bash
pnpm install
pnpm run build          # compile + lint (wireit; output goes to lib/)
pnpm run lint           # ESLint on src/ and test/
pnpm run test:only      # unit tests with coverage (c8 + mocha)
pnpm test               # compile + lint + unit tests
```

Run a single test file:

```bash
pnpm exec mocha "test/commands/pool/list.test.ts"
```

## Pre-submit checklist

1. `pnpm run lint` — no errors
2. `pnpm run test:only` — all unit tests pass (75% coverage threshold enforced by c8)
3. `pnpm run build` — compiles cleanly

NUTs (`pnpm run test:nuts`) require DevHub auth and run in CI only.

**Local NUT prerequisites**: run once per DevHub:

```bash
pnpm run setup:test-packages -- --target-dev-hub <alias>
```

If NUTs fail with `PackageVersionNotFoundError` or `SfdxProjectNotFoundError`, this setup hasn't been run or failed.

## Project structure

```
src/commands/pool/   thin CLI commands (flag parsing, output)
src/lib/             business logic — poolQuery, poolPrepare, poolFetch, poolClean, orgCreator, orgCleanup, packageInstaller
src/types/           pure TypeScript type declarations, no runtime logic
messages/            user-facing strings as Markdown (pool.<command>.md)
test/                mirrors src/; *.test.ts = unit, *.nut.ts = integration
config/              pool-example.json and project-scratch-def.json
scripts/             Node.js-only scripts (no bash; must work on Windows CI)
```

`lib/` (compiled output) is gitignored and regenerated by `pnpm run build`.
Root `sfdx-project.json` is gitignored; generated by `pnpm run setup:test-packages`.

## Key conventions

**ESM**: `package.json` sets `"type": "module"`. All local imports inside `src/` **must use `.js` extensions** even though source files are `.ts`:

```typescript
import { queryPoolOrgs } from '../../lib/poolQuery.js';
```

**Pool state** is stored on `ScratchOrgInfo` in the DevHub via two custom fields:

- `Pool_tag__c` — identifies which pool
- `Pool_allocation_status__c` — picklist: `in_progress`, `available`, `under_update`, `failed`, `assigned`

**Commands** extend `SfCommand<ResultType>` from `@salesforce/sf-plugins-core`. Load messages with:

```typescript
Messages.importMessagesDirectoryFromMetaUrl(import.meta.url);
const messages = Messages.loadMessages('pool', 'pool.<name>');
```

**User-facing strings**: never hardcode; always use message files (`messages/pool.<command>.md`).

**Errors**: `throw new SfError(messages.getMessage('error.key'))`

**Lib code**: no default exports; use `Logger.childFromRoot('name')` (not `this.log()`); accept connections/config as parameters for testability.

**Types**: use `type` not `interface`; no default exports; no runtime logic.

**Tests**: every test file uses `TestContext` + `stubSfCommandUx`; stub SOQL via `$$.fakeConnectionRequest`. See `test/commands/pool/list.test.ts` for the reference pattern.

**Commits**: conventional commits enforced by commitlint (e.g., `feat(pool:prepare): ...`, `fix(poolQuery): ...`).

**Scripts**: pure Node.js ESM, no third-party deps, use `spawnSync('sf', ...)` — never shell pipelines.
